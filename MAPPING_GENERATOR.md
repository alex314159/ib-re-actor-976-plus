# Mapping Generator

## Overview

IB Java classes are mapped to Clojure maps using a two-layer system:

- **`mapping.clj`** — core macros (`defmapping`, `defmapping-readonly`), the `Mappable` protocol, and `map->` multimethod. Also contains hand-written mappings for classes the generator cannot handle. **Edit this file for manual mappings.**
- **`generated_mappings.clj`** — auto-generated by `mapping_generator.clj` from the Java source files. **Never edit this file directly.**
- **`mapping_generator.clj`** — uses JavaParser to parse IB Java classes and emit `defmapping` declarations.

Users only need one import:

```clojure
(require '[ib-re-actor-976-plus.mapping :refer [->map map->]])
```

`mapping.clj` loads `generated_mappings.clj` automatically at the bottom of the file.

## Regenerating After an IB API Update

1. Copy new Java source files into `resources/com/ib/client/`
2. In a REPL:

```clojure
(require '[ib-re-actor-976-plus.mapping-generator :as gen])
(gen/write-generated-mappings!)
```

## IMPORTANT: Regeneration Is Not Enough

> **After regenerating, you must also review `mapping.clj` for manual mappings.**

The generator only maps classes that have:
- Private fields with matching getter **and** setter methods
- (For `defmapping`) a public no-arg constructor

Some classes are delivered by IB callbacks and are **read-only** — they have getters but no setters, and use parameterized constructors only. The generator skips these entirely. They require hand-written `defmapping-readonly` entries in `mapping.clj`.

### How to identify them

After updating the Java sources, run:

```clojure
;; See what the generator produces
(count (gen/generate-all-mappings))

;; Check a specific class
(gen/generate-mapping-for-class "HistoricalTick")  ; returns nil if skipped
```

Then grep the new EWrapper interface for any classes returned by callbacks:

```bash
grep "void " resources/com/ib/client/EWrapper.java
```

For each class type appearing in a callback signature, check whether it's in `generated_mappings.clj`. If not, add a `defmapping-readonly` to `mapping.clj`.

## How the Generator Works

1. Parses each `.java` file in `resources/com/ib/client/` using JavaParser
2. Extracts private non-static fields
3. Matches each field to a getter and setter method (IB-style parameterless `field()` preferred over `getField()`)
4. Infers translations from field types (enums → translation tables, Decimal → `:string-to-decimal`, etc.)
5. Detects nested IB objects (e.g. `DeltaNeutralContract`)
6. Emits `defmapping` (bidirectional) or `defmapping-readonly` (read-only) based on whether a public no-arg constructor exists

### Extending type inference

If the generator misses a type mapping, extend `infer-translation` in `mapping_generator.clj`:

```clojure
(or (str/includes? field-type "Types$MyNewEnum") (= field-type "MyNewEnum")) :my-new-enum
```

Then add the corresponding translation table to `translation.clj` and regenerate.
